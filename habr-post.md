# Hermes Box: собираем своего AI-ассистента для офиса за 5 минут

*Привет, Хабр! Делюсь опытом сборки полностью локального AI-стэка для малого и среднего бизнеса.*

## Проблема

ChatGPT стоит $20–30 на пользователя в месяц. Для офиса на 20 человек — $600/мес. Данные уходят в облако. Нет возможности подключить корпоративную базу знаний. А если интернет пропал — AI не работает.

Можно, конечно, развернуть Ollama локально. Но это только полдела. Нужен ещё веб-интерфейс, Telegram-бот, RAG по документам, VPN для удалённого доступа. Всё это настраивается отдельно, часами, и требует сеньора-девопса.

**Hermes Box** — open-source решение этой проблемы. Один docker-compose, 3 команды — и у вас свой AI-ассистент, который работает на вашем железе, не шлёт данные наружу и понимает ваши документы.
**RAG — почему это сложно и почему мы сделали нормально**

RAG (Retrieval-Augmented Generation) — это когда AI отвечает не из головы, а по вашим документам. Штука полезная, но дьявол в деталях.

Типичные проблемы RAG, которые мы видели в других проектах:
- Ответ есть, но без источника — непонятно, откуда ноги растут
- Источник есть, но битый — ссылка ведёт в никуда
- Если в документах нет ответа — модель выдумывает (галлюцинации)
- Векторная база требует отдельного сервера (Pinecone, Qdrant, Weaviate)
- Чанки нарезаны криво — ответ теряет контекст

У нас RAG работает так:
- В качестве векторной БД — numpy. Просто numpy-массив на диске. Никаких лишних сервисов, ничего не падает, не требует настройки
- На каждом ответе — ссылка на исходный файл. Не абстрактная, а конкретная: "взято из readme.txt"
- Если в документах ответа нет — модель честно говорит "контекст не содержит этого"
- Поддерживает русский и английский — можно спросить на любом языке, ответит по тем же документам
- Порог релевантности 0.35 — если совпадение ниже, AI не гадает, а отказывается отвечать

**В чём соль**

По отдельности все эти штуки существуют. Ollama ставится за минуту, Open WebUI — тоже. Но чтобы собрать их в работающую систему, которая реально используется в офисе, нужно:
- Настроить сеть между контейнерами
- Прикрутить Telegram-бота с белым списком
- Поднять RAG, который не глючит и не теряет контекст
- Сделать VPN, чтобы работало из дома
- Всё это задокументировать так, чтобы не-админ тоже разобрался

Вот это обычно и занимает день-два и выносит мозг. Hermes Box — это просто docker-compose.yml, который уже собран.

**Что внутри**

Стек состоит из пяти компонентов, каждый в своём контейнере:
- **Ollama** — локальный LLM-инференс. Поддерживает Qwen, Llama, Mistral, и любые модели
- **Open WebUI** — веб-чат для всей команды, с историей диалогов, загрузкой файлов, переключением моделей
- **Hermes Agent** — оркестратор с Telegram-ботом и белым списком пользователей
- **RAG Service** — чат по вашим документам (TXT, MD, PDF). Векторная БД — numpy на диске, без внешних сервисов
- **Tailscale VPN** — опциональный компонент для безопасного удалённого доступа

## Установка

Три команды:

```bash
git clone https://github.com/adcomp1971-debug/Hermes
cd Hermes && ./setup.sh
docker exec hermes-ollama ollama pull qwen2.5:7b
```

Положить документы:

```bash
cp your-docs/*.pdf documents/
curl -X POST http://localhost:8002/ingest
```

Готово. AI отвечает строго по вашим документам.

## Для кого это

- **Небольшие офисы (5–20 чел)** — RTX 3060 или просто CPU. Модель Qwen 2.5 7B, $0 за софт
- **Средний бизнес (20–100 чел)** — RTX 4090 или A100. Qwen 2.5 32B, мульти-агенты
- **Тем, кто заботится о приватности** — все данные на вашем железе, без облака

## Почему open-source

Потому что рынок self-hosted AI для среднего бизнеса практически пуст. Есть коробочные решения за $50k+ от интеграторов. Есть облачные сервисы за $20/чел. А между ними — пропасть.

Hermes Box закрывает эту дыру. Берёте своё железо (или арендуете GPU за $0.50/час на Vast.ai), ставите наш стек — и получаете AI за стоимость электричества.

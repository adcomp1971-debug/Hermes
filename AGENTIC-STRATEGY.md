---

## 1. Ключевой сдвиг: от Automation-first к Context-first

**Старая модель (умерла):**
- Жёсткие сценарии: если A → то B
- n8n / Zapier / Make — цепочки шагов
- Ломается при первом отклонении от скрипта
- Требует ручного обслуживания каждого сценария

**Новая модель (Hermes Box):**
- Агенту даётся **контекст + цель + инструменты**
- Он сам решает как достичь результата
- Адаптируется к нестандартным ситуациям
- Обучается на обратной связи

> **Ключевое:** не «запрограммируй каждый шаг», а «опиши бизнес и дай агенту задачу».

## 2. Business OS — ядро компании

### Что это

Не CRM и не база данных. **Ядро знаний и решений** — накопление лучших практик, SOP, контекста принятия решений.

### Как устроено

```
┌─────────────────────────────────────────┐
│            BUSINESS OS                   │
│                                          │
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │  SOP и   │  │  История │  │ Шаблоны│ │
│  │регламенты│  │ решений  │  │ коммун.│ │
│  └──────────┘  └──────────┘  └────────┘ │
│                                          │
│  ┌────────────────────────────────────┐  │
│  │  Внешние системы (CRM, API, почта) │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
         ↑                     ↑
    Gatekeeper            Специалисты
```

### Отличие от простого RAG

| RAG «по документам» | Business OS |
|---------------------|-------------|
| Ищет по ключевым словам | Понимает контекст задачи |
| Возвращает кусок текста | Возвращает готовое решение |
| Не помнит что сработало | Накопляет лучшие паттерны |
| Статичная база | Растёт с каждым кейсом |

### Для страхового агента

Business OS хранит:
- Условия полисов всех страховых (на иврите)
- Типовые возражения и ответы на них
- Скрипты общения с клиентами
- Историю: «в ситуации X сделали Y → клиент доволен»
- Контакты и процедуры страховых компаний (הפניקס, מגדל, הראל, כלל и т.д.)

## 3. Архитектура: Gatekeeper + Специалисты

### Общая схема

```
            ┌──────────────┐
            │  Gatekeeper   │ ← принимает запрос, маршрутизирует
            └──────┬───────┘
                   │
     ┌─────────────┼──────────────┐
     ▼             ▼              ▼
┌─────────┐ ┌──────────┐ ┌──────────────┐
│ Клиент  │ │ Документ │ │ Коммуникатор │
│ Агент   │ │ Агент    │ │ Агент        │
└─────────┘ └──────────┘ └──────────────┘
     │             │              │
     └─────────────┼──────────────┘
                   ▼
            ┌──────────────┐
            │  QA Агент    │ ← проверяет ответ перед выдачей
            └──────────────┘
```

### Роли агентов для страхового бизнеса

#### Gatekeeper
- Принимает любой запрос: голос, текст, email
- Определяет: что нужно? (инфо, действие, проблема)
- Направляет нужному специалисту
- Если не понял — уточняет

#### Клиентский агент
- Ищет клиента в CRM по имени, телефону, ID
- Показывает: действующие полисы, история платежей, статусы
- Может: создать задачу, обновить контакт

#### Документ-агент (RAG)
- Ищет по базе полисов, тарифов, условий (иврит + русский)
- Понимает специфические термины: «ביטוח צד ג׳», «מקיף», «חבילת בסיס»
- Может сравнить условия разных страховых

#### Коммуникатор
- Помогает составить письмо/сообщение на иврите
- Учитывает тональность: официально / дружески / срочно
- Проверяет: все ли обязательные пункты указаны

#### QA-агент
- Проверяет ответ перед отправкой клиенту
- Контролирует: нет ли ошибок, не пропущены ли детали
- Если сомневается — запрашивает подтверждение человека

## 4. Продуктовая стратегия

### Текущее состояние

Сейчас Hermes Box — это **инфраструктура**: Docker Compose с Ollama, WebUI, Hermes Agent, Guardrails. Клиент получает сервер и говорит «ну и что мне с этим делать?».

### Целевое состояние

**Hermes Box = AI-сотрудник, готовый к работе в твоём бизнесе.**

Клиент:
1. Создаёт компанию (1 клик)
2. Выбирает вертикаль: «Страховой агент», «Адвокат», «Логист»
3. Загружает документы (PDF, Excel, Word)
4. Подключает внешние системы (WhatsApp, почта, CRM)
5. Получает готового AI-сотрудника

### Модель монетизации

| Уровень | Цена | Что включено |
|---------|------|-------------|
| **Basic** | ₪299/мес | 1 агент, базовый RAG, Telegram/WhatsApp |
| **Business** | ₪599/мес | 3 агента (Gatekeeper + 2 спеца), CRM-интеграция |
| **Premium** | ₪999/мес | Полная архитектура, 5+ агентов, приоритетная поддержка |

**Цена за результат, а не за фичи.** Клиент платит не за «доступ к LLM», а за то что бот реально помогает вести дела.

### Первая вертикаль: Страховой агент (סוכן ביטוח)

Почему:
- Высокая ценность ошибки → готовы платить за качество
- Много документов на иврите → RAG критичен
- Одиночки или малые команды → нужен помощник, не замена
- Регуляторная отчётность → AI-агент как память и контроль

## 7. Ключевые принципы

1. **Не автоматизация, а делегирование.** Агент не выполняет скрипт — он решает задачу.
2. **Ядро знаний важнее модели.** Лучшая модель без контекста бесполезна. Средняя модель с хорошей Business OS — золото.
3. **Оплата за результат.** Если бот помог закрыть сделку — это ценность. Если просто ответил — базовая подписка.
4. **Контроль без микро-менеджмента.** QA-агент + человеческое подтверждение для критических действий.
5. **Каждая вертикаль — продукт.** Не «настрой под себя», а «выбери свой бизнес и работай».
